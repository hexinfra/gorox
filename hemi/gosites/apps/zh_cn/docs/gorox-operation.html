构建与安装

在构建 Gorox 前，请确保您的操作系统符合以下要求：

  • Linux 内核版本 >= 3.9
  • FreeBSD >= 12.0
  • 苹果 macOS >= Catalina
  • 微软 Windows >= 10

以及您的 CPU 架构符合以下要求：

  • AMD64，也称为 x64，x86-64，Intel 64
  • ARM64，也称为 AArch64
  • RISCV64，也称为 RV64，RISC-V 64
  • LOONG64，也称为 LA64，LoongArch 64

然后，请确保您安装了 Go >= 1.19 版本：

shell> go version

构建

进入 Gorox 目录，使用以下命令构建 Gorox：

shell> go build

如果报错，请设置环境变量 CGO_ENABLED=0 再试一次：

shell> go env -w CGO_ENABLED=0
shell> go build

安装

在您构建成功，或者下载了二进制版本后，直接把 gorox 所在的整个目录移动到您想要存放的位置，即可。

运行模式

Gorox 具备两种运行模式，一种是充当服务器用的服务器模式，另一种是充当客户端用的客户端模式。

服务器模式

正如您所希望的那样，Gorox 可以以服务器进程的方式运行。如果您加了“-daemon”选项，它可以进入后台运行，如果不加，则默认在前台运行。

客户端模式

在客户端模式下，Gorox 充当一个控制客户端，用于控制另一个在服务器模式下运行的 Gorox 进程。

启动

以前台模式启动 Gorox 服务器：

shell> ./gorox serve

以后台模式启动 Gorox 服务器：

shell> ./gorox serve -daemon

以后台模式启动 Gorox 服务器后，可以运行以下命令检查 leader 进程和 worker 进程是否启动成功：

shell> ./gorox pid

退出

运行一个客户端模式的 Gorox，可以告知当前正在运行的处于服务器模式的 Gorox 进程退出运行：

shell> ./gorox quit

重启

通常来说，我们都是为了重启 worker 进程。使用以下命令重启 worker 进程：

shell> ./gorox rework
