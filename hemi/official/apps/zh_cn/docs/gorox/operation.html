Gorox 操作手册

本文描述如何使用 Gorox。

如果您是一位开发者，在阅读完本文后，还希望学习如何使用 Gorox 进行 Web 应用或 RPC 服务开发，那么请继续参考 Gorox 开发者手册。

编译与安装

在编译 Gorox 前，请确保您的运行平台符合以下要求：

  • Linux 内核版本 >= 3.9，CPU 架构为 AMD64 或 ARM64；
  • FreeBSD >= 12.0，CPU 架构为 AMD64；
  • 苹果 macOS >= Catalina，CPU 架构为 AMD64 或 ARM64；
  • 微软 Windows >= 10，CPU 架构为 AMD64。

然后，请确保您安装了 Go >= 1.19 版本。

编译

shell> go build

安装

在您编译成功，或者下载了二进制版本后，直接把 gorox 所在的整个目录移动到您想要存放的位置，即可。

运行模式

Gorox 具备两种运行模式，一种是充当服务器用的服务器模式，另一种是充当客户端用的控制器模式。

服务器模式

正如您所希望的那样，Gorox 可以以服务器进程的方式运行。如果您加了“-daemon”选项，它可以进入后台运行，如果不加，则默认在前台运行。

控制器模式

在控制器模式下，Gorox 充当一个控制客户端，用于控制另一个在服务器模式下运行的 Gorox 进程。

启动

以前台模式启动 Gorox：

shell> ./gorox serve

以后台模式启动 Gorox：

shell> ./gorox serve -daemon

停止

运行一个控制器模式的 Gorox，可以告知当前正在运行的处于服务器模式的 Gorox 进程停止运行：

shell> ./gorox quit

重启

通常来说，我们都是为了重启 worker 进程。使用以下命令重启 worker 进程：

shell> ./gorox rework
